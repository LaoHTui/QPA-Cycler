# save2docx功能介绍

## 一、脚本概述

`save2docx.py`是 QPA-Cycler 准周期分析程序的配套报告生成工具，主要用于将准周期分析的结果（包括真实数据和模拟数据的分析结果）生成为规范、美观的 Word 格式报告文档，实现分析数据、可视化图表、参数配置的整合输出，方便用户查看和归档分析结果。

## 二、核心功能模块

### 2.1 状态管理模块

该模块负责程序运行状态的保存与加载，支持断点续跑相关的状态记录：

1. **状态保存**：通过`save_state`函数将程序的运行状态（如处理进度、已处理文件、分析参数等）以 JSON 格式保存到指定文件中，确保程序中断后可以恢复之前的运行状态。

2. **状态加载**：通过`load_state`函数从指定的 JSON 文件中加载之前保存的运行状态，获取历史处理信息和参数配置。

### 2.2 图片处理模块

该模块用于处理分析过程中生成的可视化图表，实现图片的整理和展示：

1. **占位图片生成**：当某个算法的可视化图片缺失时，通过`create_placeholder_image`函数生成带有提示文本的占位图片，保证文档布局的完整性。

2. **图片分组排序**：通过`get_sorted_images`函数将图片按数字前缀分组，并按照指定的算法顺序（LSP、DCF、JV、WWZ）对每组内的图片进行排序，确保图表展示的规范性。

3. **图片布局展示**：将排序后的图片以 2×2 四宫格的布局插入到 Word 文档中，同时为每个图片添加对应的算法名称标题。

### 2.3 Word 文档生成模块

该模块是脚本的核心模块，负责生成完整的 Word 报告文档，包含以下功能：

1. **页面布局设置**

    - 自定义页眉：左侧显示程序名称，右侧显示当前日期和时间。

    - 自定义页脚：添加居中的页码，方便文档的翻阅和引用。

    - 页面边距设置：调整文档的上下左右边距，优化内容展示效果。

2. **表格生成**

    - **结果表格**：将分析得到的周期检测数据整理成三线表格式的表格，展示每个数据源的检测周期及误差信息。

    - **参数表格**：将分析使用的参数配置（包括自动模式分析参数、模拟参数等）以表格形式展示，方便用户查看分析的配置信息。

    - 表格样式自定义：支持无边框表格、三线表等样式，同时可以设置表格的行高、列宽、单元格边距等属性。

3. **内容组织**

    - 文档标题设置：支持自定义报告标题，居中显示并设置字体样式。

    - 书签添加：为每个图片分组添加书签，方便快速定位文档内容。

    - 详细说明添加：为每个分组添加周期检测的详细说明文本，包括周期数值、误差、检测结果说明等。

### 2.4 数据处理模块

该模块负责分析数据的整理和处理，为报告生成提供结构化的数据：

1. **周期数据重组**：通过`reorganize_period_data_list`函数将原始的周期检测数据整理成适合表格展示的结构化数据，将多个周期的检测结果转换为多列表格形式。

2. **标签生成**：通过`get_labels`函数从原始数据中提取周期检测的详细信息，生成可读的说明文本，用于在报告中展示每个数据源的周期检测详情。

3. **模拟数据准确度分析**：针对模拟数据的分析结果，通过`analyze_period_accuracy`等函数分析周期检测的准确度，生成性能分析报告，包括检测率、误报率、准确率等统计指标。

## 三、使用方式

### 3.1 真实数据报告生成

调用`save2docx`函数生成真实数据的分析报告，需要指定以下参数：

- `data_path`：分析结果数据的存储路径，包含运行状态文件和图片文件。

- `state_dict_filename`：运行状态文件的名称（不含后缀）。

- `json_params_filename`：参数配置文件的名称（不含后缀）。

- `title`：报告的标题，默认为 "Report on the Results of the Quasi Periodic Analysis Program for Fermi Data"。

- `docx_output_path`：Word 文档的输出路径，默认为当前目录。

- 其他可选参数：图片宽度、表格行高、表格单元格宽度等，用于自定义文档布局。

### 3.2 模拟数据报告生成

调用`save2docx_sim`函数生成模拟数据的分析报告，该函数会额外生成性能分析报告，参数与真实数据报告生成的参数类似，同时会在报告中添加模拟数据的检测准确度统计内容，包括文件统计、周期检测结果、性能指标等信息。

## 四、输出文档结构

生成的 Word 报告文档包含以下内容结构：

1. **报告标题**：居中显示的报告主标题，可自定义。

2. **结果表格**：以三线表形式展示所有数据源的周期检测结果，包含数据源名称、检测周期、周期误差等信息。

3. **参数表格**：展示分析使用的参数配置，包括自动模式分析参数、模拟参数等。

4. **图片组**：按分组展示不同算法的可视化图表，每个分组以 2×2 四宫格布局展示 LSP、DCF、JV、WWZ 四种算法的图表，同时添加分组标题和书签。

5. **详细说明**：为每个分组添加周期检测的详细说明文本，展示每个周期的数值、误差和检测结果说明。

6. **性能分析（仅模拟数据报告）**：展示模拟数据的检测性能统计，包括检测率、误报率、准确率等指标。

## 五、依赖说明

运行该脚本需要安装以下 Python 库：

1. `python-docx`：用于创建和编辑 Word 文档。

2. `pandas`：用于数据处理和表格生成。

3. `Pillow（PIL）`：用于图片处理和占位图片生成。

4. `json`：用于状态文件的读写。

5. `os`、`re`、`datetime`等 Python 内置库：用于文件操作、正则匹配、日期处理等。
