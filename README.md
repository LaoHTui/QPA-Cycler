# QPA-Cycler：多方法准周期分析自动化流水线

**QPA-Cycler** (Quasi-Periodicity Analysis Cycler) 是一款专为时间序列数据（特别是天文光变曲线等**非均匀采样数据**）设计的自动化周期检测工具。它集成了多种经典的周期分析算法，实现了从数据预处理到自动化生成研究报告的全流程覆盖。

## 核心架构：自动化周期检测流程
该工具遵循以下标准处理流水线：
**数据读取 → 数据预处理 → 模式与算法选择 → 周期性分析 → 结果可视化 → 自动化文档导出**


## 🚀 功能特性

1.  **多算法集成**：内置 DCF、Jurkevich (PDM)、Lomb-Scargle (LSP)、WWZ (加权波谱分析) 及谐波分析等主流周期检测算法。
2.  **非均匀采样支持**：针对天文观测等存在间隙或采样不均的数据进行了算法优化。
3.  **高度程序自动化**：支持通过配置文件（`config.json`）批量处理多个数据源，算法参数直观可调。
4.  **容错与断点续传**：
    *   **分钟级自动备份**：程序运行期间每分钟更新一次中间状态。
    *   **状态恢复**：若程序意外中断，可通过读取状态文件从断点处继续运行。
5.  **专业化结果输出**：
    *   **一键制图**：生成光变曲线图、功率谱图、相位折叠图等。
    *   **Word报告导出**：通过 `save2docx.py` 将分析参数、中间图表、计算结果自动整理为结构化的 Word 文档。
6.  **仿真数据生成**：内置模拟信号生成工具，便于测试算法精度与可靠性。

## 📖 使用指南

### 1. 准备数据
将您的时间序列数据存放在指定文件夹中，支持 `CSV` 或 `TXT` 格式（通常包含时间、通量、误差三列）。
*详细数据要求请参考：`docs/README_getdata_zh.md`*

### 2. 配置参数
编辑 `config.json` 文件。主要配置包括：
*   `folder_path`: 输入数据路径。
*   `output_path`: 结果保存路径。
*   `algorithms`: 选择要启用的算法及其参数（如频率步长、搜索范围等）。
*   `rerun`: 是否重新计算（若为 `False` 则尝试从状态文件恢复）。

### 3. (可选) 生成模拟数据测试
如果您想测试程序，可以运行：
```bash
python gen_sumulated_data.py
```
这将在指定路径生成带有已知周期和噪声的模拟信号。

### 4. 运行主分析程序
执行主程序开始进行周期分析：
```bash
python main.py
```
程序运行时将实时显示计算进度，并周期性保存状态至 `Running_data` 文件夹。

### 5. 导出分析报告
分析完成后，运行导出程序：
```bash
python save2docx.py
```
该操作会读取 `state` 文件和参数信息，并生成一个包含所有计算结果图表的专业 Word 文档（`Result-Report.docx`）。

## 📂 项目结构说明

*   `main.py`: 核心运行程序。
*   `save2docx.py`: 报告生成模块。
*   `gen_sumulated_data.py`: 信号模拟模块。
*   `config.json`: 全局参数配置文件。
*   `docs/`: 详细的算法原理、参数设置及各模块说明文档。
*   `example/`: 包含可供参考的输入示例及输出结果样本。

## 📊 结果查看
运行完成后，`output_path` 目录下将包含：
- **`Light_Slot/`**: 输入数据的原始光变曲线图。
- **`Running_data/`**: 包含 `state` 状态文件和运行时的配置文件副本。
- **周期分析图表**: 
    *   `*_LSP.png`: Lomb-Scargle 周视图。
    *   `*_WWZ.png`: WWZ 时频分析图。
    *   `*_periodic_*.png`: 各算法的最佳周期拟合及相位折叠图。
